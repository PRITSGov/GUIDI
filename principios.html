Botones que requieran presionarse deben tener una altura de 70 px máx / 60 px min. El largo va a depender del texto o función que dicho botón tenga designado. Debe existir una separación entre cada botón. Guía

  • Es necesario utilizar los botones primarios para las acciones más importantes que queremos que usuario haga, como Descargar, Someter y Seguir.
  • Para una acción secundaria o menos importante podemos utilizar los botones secundarios para que el usuario entienda la jerarquía.

Cuando utilizar el componente de la tarjeta

Colecciones de contenidos relacionados. Cuando necesite presentar una colección de grupos de contenidos relacionados, como artículos, secciones de un sitio web e información sobre la agencia o municipio.

Una tarjeta suele ser corta y ofrece un punto de entrada vinculado a otros detalles, más que los detalles completos en sí. Cuando se intenta meter en una tarjeta demasiado contenido, hasta el punto de que la tarjeta empieza a ser demasiado ancha o demasiado larga, pierde su metáfora original, ya que deja de parecer una tarjeta.

No utilice el componente de la tarjeta sólo para la decoración. Utilice el componente de la tarjeta para las tarjetas, no para cualquier tipo de contenido que esté diseñado para tener un borde alrededor.

Carrusel y Galerías

Anuncios, promociones, ofertas, etc. (relacionados o no a la agencia) deberán ser todos del mismo tamaño y ubicados dentro de un carrusel que no ocupe un área mayor de una columna. Esta sección es para elementos que no requieran una página adicional.

Las galerías de fotos se trabajarán a modo de carrusel, esto evitará la saturación de imágenes en los portales cuando los temas sean parecidos. En casos de galerías de fotos de eventos particulares, es responsabilidad de la Agencia hacer un escogido conciso de las imágenes a ser publicadas para así hacer que la experiencia del usuario sea más agradable.

Muchos de los problemas del carrusel surgen del hecho de que a menudo avanzan automáticamente, sin que el usuario haga nada. Así que hay una solución sencilla: no hacer la reproducción automática.

Permitir que el usuario tenga el control del movimiento del carrusel. Ya es común para el usuario ver íconos de flechas y puntos para indicar la navegación.

El texto por cada plantilla debe ser corto y claro.

Limitar la cantidad de plantillas para no sobre cargar al usuario al momento de navegar la página.

Importante mantener todo un mensaje en una sola plantilla. No crear un continuidad entre plantillas.

Multimedia

Imágenes o videos de gran importancia que se deseen utilizar para resaltar algún tema, pueden sobrepasar el sistema de columnas establecido en las guías, siempre y cuando se tome en consideración el ajuste de las pantallas (desktop vs mobile).

Guías - Imágenes

  • Para ayudar al usuario es necesario que las fotos vayan de acuerdo con el contexto.
  • Evite hacer que el usuario busque el significado en la imagen y asegúrese de que se le transmite un concepto claro de forma memorable.
  • Las imágenes humanas son una forma muy eficaz de conseguir que los usuarios se comprometan y se sientan más conectados.

Todo video tiene que tener la capacidad de activarse dentro del mismo portal y no salir a páginas externas. Ningún video debe empezar de manera automática, excepto que el audio esté deshabilitado.

Guías - Videos

  • No hay que depender demasiado del vídeo para transmitir información. No todos los usuarios miran y atienden los videos.
  • Configura el audio como desactivado por defecto con la opción de activarlo.
  • Evita las presentaciones largas, es esencial empezar con fuerza.
  • Mantener los vídeos lo más cortos posible.
  • Los contenidos de vídeo solo son útiles si los usuarios tienen control sobre ellos.
  • No se olvide de la accesibilidad. Ofrezca una forma alternativa de acceder a los contenidos.

Procesos y Formas

Pasos

Todo proceso que se necesite completar dentro del portal requiere una explicación concisa de los pasos a seguir para así orientar de manera eficiente al cuidadano. De esta manera se evita la confusión en los pasos a seguir para completar formularios.

1

Paso #1 Pensar en el Usuario

No existe una manera universal para trabajar los pasos para formas o acciones que se necesitan completar. Lo importante es pensar en las formas que los usuarios interactúan con otras plataformas o aplicaciones.

2

Paso #2 Tener los pasos claros.

El lenguaje tiene que ser claro y sencillo al momento de explicar cada paso que es necesario para completar la acción. No podemos crear elementos esperando una interacción del usuario sin tener las debidas instrucciones.

  • Establecer para el usuario la cantidad de pasos que necesitan ser completados.
  • Definir claramente la información o acción que se necesita del usuario.
  • Dar un aproximado de cuanto tiempo se puede tardar el usuario en completar la acción.
  • Informas claramente al usuario en que paso se encuentra durante todo el proceso.
3

Paso #3 Mejor experiencia

El usuario tiene más determinación y seguridad para terminar la acción. También permite la progresión lógica de la acción o forma que se está completando. Por último ayuda atener una mejor impresión de la página o agencia.

Pasos y pantallas múltiples.

Pasos y pantallas múltiples. Utilice el indicador de pasos cuando el usuario esté trabajando en un formulario o proceso que abarque varias páginas diferentes que puedan organizarse en tres o más pasos o capítulos de alto nivel. Esto ayuda a los usuarios a entender la progresión de los formularios largos y reduce la incertidumbre mientras completan cada pantalla.

Progresión lineal. El indicador de pasos está diseñado para complementar la navegación estándar hacia atrás/siguiente en una secuencia lineal, no para ser una navegación propia.

Acordeones

Se utilizarán acordeones multi-seleccionables en los casos que se necesite proveer información extensa sobre múltiples temas. Estos acordeones permiten expander o reducir el texto, de esta manera darle al usuario más control de la información que va a consumir, sin la necesidad de que se vea cargado el portal.

Cuando usar el acordeón:

  • Si los usuarios solo necesitarán algunas piezas específicas de contenido dentro de una página.
  • Si solo tiene un pequeño espacio para mostrar una gran cantidad de contenido.

Cuándo considerar otra cosa:

  • Si los usuarios necesitan ver la mayor parte o toda la información de una página. Utilice en su lugar un texto bien formateado.
  • Si no hay suficiente contenido para justificar la condensación. Los acordeones aumentan la carga cognitiva y el coste de la interacción, ya que los usuarios tienen que tomar decisiones sobre los encabezados en los que deben hacer clic.

Orientación sobre la usabilidad:

  • Haga que todo el header(cabecera) sea seleccionable. Permita a los usuarios hacer clic en cualquier parte del área del header(cabecera) para ampliar o contraer el contenido; un blanco más grande es más fácil de manipular. Deje espacio suficiente a los elementos interactivos.
  • Asegúrese de que los elementos interactivos dentro de la región del elemento estén lo suficientemente lejos de los encabezados para que los usuarios no provoquen accidentalmente un colapso. (La distancia exacta depende del dispositivo).

Orientación sobre la funcionalidad:

  • Si solo tiene 3-4 categorías, pero los datos son demasiados, entonces hay posibilidades de que los acordeones sean una mala opción para utilizar, ya que no se puede ver tanto las categorías como los datos en una sola ventana. Tendrá que añadir un desplazamiento para el usuario.
  • ¿Cuál es la arquitectura que sigue su sitio web en todas las páginas? Esta pregunta le aclarará si debe utilizar pestañas o acordeones. Si su sitio web tiene una navegación vertical, debería utilizar acordeones para mantener el mismo patrón y para que el usuario se sienta cómodo en todo el sitio web (pero, de nuevo, después de considerar otros factores).
  • Si son demasiado largos, entonces los acordeones son una mejor opción en lugar de las pestañas. Los acordeones colocados en un espacio horizontal tienen demasiado espacio para acomodar opciones largas y muchos datos, mientras que en las pestañas si elige nombres largos como etiquetas, ¡se verán más o menos como acordeones solamente!

Tabs y Menú Lateral

  • Relación
  • Espacio
  • Consistencia

Información relacionada

En primer lugar, debe establecer cómo se dividirá el contenido del sitio, lo que implica agrupar información relacionada, imágenes y otros elementos bajo el mismo término general. Como regla general, solo debe utilizar pestañas cuando se trate de al menos tres y no más de nueve o diez secciones distintas. Una vez que haya establecido en qué grupos encajan todos los contenidos disponibles, debe asignar una etiqueta de pestaña lógica e inequívoca a cada categoría y clasificarlos en un orden lógico.

Espacio y contenido

A continuación, hay que disponer todas estas etiquetas en horizontal, bordeando unas con otras, para reducir el espacio que debe recorrer el usuario al pasar de una pestaña a otra, y para garantizar que haya espacio suficiente para que quepa toda la información necesaria en cada pestaña. La convención habitual para las pestañas de navegación es asegurarse de que la barra de pestañas abarque el ancho superior de la pantalla, ya que esto ayuda a establecer el primer nivel en el marco visual, con las pestañas de navegación sirviendo como el nivel superior y proporcionando los canales más amplios para explorar.

Consistencia a través de la página

Asegúrese de que se mantiene el mismo diseño en todas las páginas, para permitir un movimiento libre y fácil por el contenido del sitio y evitar errores. Es más probable que los usuarios cometan errores de selección si las pestañas cambian constantemente de lugar. Además, un diseño incoherente obliga a los usuarios a relacionarse con el despliegue visual de forma consciente, en lugar de confiar en la experiencia para guiar sus interacciones.

Utilidad de Menú Lateral

Este tipo de módulos se utilizarán cuando la Agencia necesita proveer al ciudadano un catálogo extenso de documentos, formularios, leyes, guías, cartas, memorandos, órdenes administrativas, etc.

Casos extensos

En estos casos es extremadamente importante proveer un sistema de navegación o búsqueda que facilite el encontrar estos documentos. Con el uso de paginación (según se requiera) se evitarán el uso de listas interminables

Barra de Búsqueda

Cuando los documentos o información son complicados y extensos es una buena practicar establecer un sistema de búsqueda donde el usuario puedo buscar directamente la información para mejor la experiencia del usuario.

Orden por Categorías

Para el uso de tablas, tarjetas de información o documentos es bueno que el usuario pueda ordenar los elementos que está buscando. Mientras más fácil se pueda llegar al objetivo que tiene el usuario mejor será su experiencia y respuesta.

Principios

El filtrado como concepto cumple dos principios a la vez:

  • Principio de elección - es decir, cuando un usuario tiene una tarea concreta, debemos proporcionarle un montón de opciones significativas para que la lleve a cabo.
  • Principio de clasificación múltiple: significa que debemos dar a nuestros usuarios la oportunidad de navegar por nuestro contenido de diferentes maneras.
