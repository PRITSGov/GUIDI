Botones y Navegación

Botones que requieran presionarse deben tener una altura de 70 px máx / 60 px min. El largo va a depender del texto o función que dicho botón tenga designado. Debe existir una separación entre cada botón. Debe distinguirse con el uso de colores cuál es el botón que se presionaría para un próximo paso La función de cada botón se distinguirá con el uso de colores, indicando la funcionalidad de cada uno.

Guía

  • Es necesario utilizar los botones primarios para las acciones más importantes que queremos que usuario haga, como Descargar, Someter y Seguir.
  • Para una acción secundaria o menos importante podemos utilizar los botones secundarios para que el usuario entienda la jerarquía.
  • Es importante asegurarnos que los botones parezcan seleccionables.
  • Para una mejor experiencia de usuario y accesibilidad es importante crear interacción de :hover y :focus es los botones.
  • Los botones deben mostrar un estado de :focus visible cuando los usuarios se dirijan a usando "tab" ellos.

Colecciones de contenidos relacionados. Cuando necesite presentar una colección de grupos de contenidos relacionados, como artículos, secciones de un sitio web e información sobre la agencia o municipio.

Una tarjeta suele ser corta y ofrece un punto de entrada vinculado a otros detalles, más que los detalles completos en sí. Cuando se intenta meter en una tarjeta demasiado contenido, hasta el punto de que la tarjeta empieza a ser demasiado ancha o demasiado larga, pierde su metáfora original, ya que deja de parecer una tarjeta.

Detalles Sencillos

27/06/2021

Utilice un estilo sencillo. Evite el skeumorfismo que distraiga. No incluya ningún estilo de tarjeta que llame demasiado la atención sobre la metáfora de una tarjeta de papel, como pliegues, bordes doblados o textura del papel. No utilice el componente de la tarjeta sólo para la decoración. Utilice el componente de la tarjeta para las tarjetas, no para cualquier tipo de contenido que esté diseñado para tener un borde alrededor.

Carrusel y Galerías

Anuncios, promociones, ofertas, etc. (relacionados o no a la agencia) deberán ser todos del mismo tamaño y ubicados dentro de un carrusel que no ocupe un área mayor de una columna. Esta sección es para elementos que no requieran una página adicional.

Las galerías de fotos se trabajarán a modo de carrusel, esto evitará la saturación de imágenes en los portales cuando los temas sean parecidos. En casos de galerías de fotos de eventos particulares, es responsabilidad de la Agencia hacer un escogido conciso de las imágenes a ser publicadas para así hacer que la experiencia del usuario sea más agradable.

Muchos de los problemas del carrusel surgen del hecho de que a menudo avanzan automáticamente, sin que el usuario haga nada. Así que hay una solución sencilla: no hacer la reproducción automática.

Permitir que el usuario tenga el control del movimiento del carrusel. Ya es común para el usuario ver íconos de flechas y puntos para indicar la navegación.

El texto por cada plantilla debe ser corto y claro.

Limitar la cantidad de plantillas para no sobre cargar al usuario al momento de navegar la página.

Importante mantener todo un mensaje en una sola plantilla. No crear un continuidad entre plantillas.

    let table, rows, switching, i, x, y, shouldSwitch;
    table = document.getElementById("myTable");
    switching = true;
    /*Make a loop that will continue until
    no switching has been done:*/
    while (switching) {
      //start by saying: no switching is done:
      switching = false;
      rows = table.rows;
      
      /*Loop through all table rows (except the
      first, which contains table headers):*/
      for (i = 1; i < (rows.length - 1); i++) {
      //start by saying there should be no switching:
      shouldSwitch = false;
      var orderIndex = $("#order").val()
      /*Get the two elements you want to compare,
      one from current row and one from the next:*/
      x = rows[i].getElementsByTagName("TD")[orderIndex];
      y = rows[i + 1].getElementsByTagName("TD")[orderIndex];
      //check if the two rows should switch place:
      if (x.innerHTML.toLowerCase() > y.innerHTML.toLowerCase()) {
        //if so, mark as a switch and break the loop:
        shouldSwitch = true;
        break;
      }
      }
      if (shouldSwitch) {
      /*If a switch has been marked, make the switch
      and mark that a switch has been done:*/
      rows[i].parentNode.insertBefore(rows[i + 1], rows[i]);
      switching = true;
      }
